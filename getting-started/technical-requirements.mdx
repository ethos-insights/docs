Review these requirements before deploying Trek to ensure a smooth experience for students and teachers.

## Supported devices

| Device type | Minimum specs |
| --- | --- |
| Chromebook | Chrome OS 107 or later; 4 GB RAM recommended |
| Windows laptop/desktop | Windows 10 or later; 4 GB RAM; dual-core processor |
| Mac | macOS 12 (Monterey) or later; 4 GB RAM |
| iPad | iPadOS 15 or later |
| Android tablet | Android 10 or later; 4 GB RAM |

<Note>
  Trek is optimized for Chromebooks and is commonly deployed in 1:1 Chromebook environments.
</Note>

## Supported browsers

- **Google Chrome** (version 107\+) — recommended
- **Microsoft Edge** (version 107\+)
- **Safari** (version 15\+)
- **Mozilla Firefox** (version 105\+)

Browser extensions that block JavaScript or third-party cookies may interfere with Trek. Students should use a managed browser profile where possible.

## Network requirements

### Bandwidth

- Minimum: 1 Mbps per concurrent user
- Recommended: 5 Mbps per concurrent user for rich media content

### Allowlisted domains

Ensure the following domains are allowlisted in your network firewall and content filter:

```text
*.treklearning.com
*.google.com (if using Google SSO)
*.googleapis.com
*.gstatic.com
fonts.googleapis.com
fonts.gstatic.com
```

### Ports

- HTTPS (TCP 443) — required
- HTTP (TCP 80) — redirects to HTTPS; can be blocked if needed

## Accessibility

Trek meets WCAG 2.1 AA standards. Built-in features include:

- Screen reader support (NVDA, JAWS, VoiceOver)
- Keyboard-only navigation
- High-contrast mode
- Adjustable text size

## IT checklist

- Devices meet minimum hardware requirements
- Supported browser installed and up to date
- Trek domains allowlisted in firewall and content filter
- Student accounts rostered (see [Student Rostering Requirements](/getting-started/student-rostering-requirements))
- SIS or SSO integration configured (see [School System Integrations](/getting-started/school-system-integrations))
